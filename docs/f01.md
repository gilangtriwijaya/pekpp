F01 — Pengisian Mandiri UPP
=================================

Ringkasan
---------
F01 adalah modul pengisian kinerja UPP berbasis indikator dan pertanyaan dinamis.
Data sumber kebenaran berada di model/ tabel: `F01Pengisian`, `F01IndikatorNilai`, `F01IndikatorBukti` (tidak diubah secara konseptual).

Prinsip utama
------------
- 1 UPP × 1 periode = 1 `F01Pengisian` (unik)
- Status: `draft`, `submitted`, `rolled_back` (ENUM)
- Setelah submit → snapshot terkunci (tidak boleh diubah kecuali rollback resmi)
- Jawaban mentah disimpan terpisah dari `F01IndikatorNilai` jika diperlukan; F02 hanya membaca `F01IndikatorNilai` & `F01IndikatorBukti`
- Relasi `user ↔ upp` hanya melalui `user_upp`

Arsitektur komponen
-------------------

Layer dan tanggung jawab

- Controller
  - `F01PengisianController` — menampilkan daftar dan form, membuat entri pengisian jika belum ada, status view, submit(finalize)
  - `F01JawabanController` — autosave jawaban (AJAX), store/update per pertanyaan, bulk save
  - `F01BuktiController` — upload / delete bukti, relasikan ke `F01IndikatorNilai`

- Service
  - `F01ScoringService` — menghitung nilai indikator (ekstrak logika dari controller), finalisasi pengisian

- Validation
  - `SubmitF01Request`, `StoreJawabanRequest`, `StoreBuktiRequest`

- Policy / Middleware
  - `F01PengisianPolicy` — memastikan hanya user dengan `user_upp.peran` = `operator|penyusun` yang bisa edit; pusat/admin read-only

- UI
  - Blade views: `resources/views/f01/index.blade.php`, `resources/views/f01/show.blade.php`, partial `_indicator.blade.php`
  - JS: `public/js/f01.js` (autosave, disable on locked, render dynamic questions)

Data flow & kontrak antar-layer
------------------------------

1. User membuka `GET /f01` → `F01PengisianController@index`
   - Controller memeriksa user_upp, periode aktif, mencari/membuat `F01Pengisian` (status `draft`)
   - Return view `f01.show` dengan model `F01Pengisian` dan relasi `indikatorNilai` dan konfigurasi pertanyaan

2. Halaman render indikator & pertanyaan dinamis
   - Blade mem-iterate indikator; setiap indikator memuat pertanyaan (type, options, bobot)
   - JS membuat autosave: ketika jawaban berubah, kirim `POST /f01/jawaban` (storeOrUpdate) -> controller menyimpan jawaban mentah dan mengembalikan preview nilai indikator

3. Scoring
   - `F01JawabanController` tidak menghitung final score; hanya dapat memanggil `F01ScoringService->hitungNilaiIndikator()` untuk preview nilai sementara
   - Final scoring dijalankan oleh `F01ScoringService->finalisasiPengisian()` saat `submit()` di `F01PengisianController`
   - `F01ScoringService` menulis hasil ke `F01IndikatorNilai`

4. Upload bukti
   - `F01BuktiController@store` menyimpan file ke Storage (local/S3), membuat record `F01IndikatorBukti` dan mengikat ke `F01IndikatorNilai`

5. Submit
   - `F01PengisianController@submit` memvalidasi semua persyaratan (jawaban wajib, bukti wajib), memanggil `finalisasiPengisian()` di service, merubah status menjadi `submitted`, dan menulis snapshot timestamp

Hal-hal penting dan perhatian implementasi
----------------------------------------

- Jangan mengubah struktur konseptual model yang sudah ada; hanya tambahkan relasi Eloquent bila perlu (hasMany, belongsTo).
- Gunakan DB transaction pada operasi tersusun (autosave bulk, finalisasi, upload bukti yang menulis beberapa tabel).
- Autosave harus idempotent dan toleran terhadap konflik (lock optimistic) — gunakan `updated_at` untuk deteksi perbedaan.
- Kirim preview nilai dari service, jangan dihitung di client.
- Setelah `submitted`, semua endpoint yang mengubah data harus menolak dengan 403.
- Sediakan endpoint read-only untuk F02: query `F01Pengisian::with('indikatorNilai.bukti')->where(...)`.

Validasi dan aturan bisnis utama
--------------------------------

- Semua pertanyaan wajib harus terisi sebelum submit
- Bukti yang ditandai wajib harus ter-upload
- Indikator tanpa nilai (NULL) menghalangi submit
- Hanya user dengan `user_upp.peran` = `operator|penyusun` dapat edit
- Admin pusat: baca saja

Checklist testing
-----------------

1. Satu UPP tidak bisa memiliki >1 `F01Pengisian` untuk periode yang sama.
2. Autosave menyimpan jawaban per-pertanyaan.
3. Preview nilai indikator terlihat setelah autosave.
4. Submit melakukan finalisasi, menulis nilai ke `F01IndikatorNilai` dan mengunci pengisian.
5. F02 dapat membaca `F01IndikatorNilai` dan `F01IndikatorBukti` tanpa akses ke jawaban mentah.
